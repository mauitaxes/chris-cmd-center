## The five changes (confirmed design decisions)

### 1. Reset engine overhaul — fixes "daily reset not happening" + "morning routine not resetting"
These two reported bugs share one root cause: the daily reset isn't firing. New design:

- **Trigger:** on every app load, compare today's date to `lastResetDate` stored in State. If different → run the reset. (This replaces dependence on a midnight timer, which misses when the app is closed.)
- **Reset sequence — order matters:**
  1. If no saved report exists for the prior day, **generate it now** from the ending state (safety net — never lose a day).
  2. Save the prior day's report to Notion **in the background** — do NOT await it before rendering.
  3. Clear the day's state locally **immediately** (routine checkmarks, etc.).
  4. Set `lastResetDate = today`.
  5. **First run of the day only:** route to the Routine page (see change 3).
- **Keep** the existing End-of-Day report button and the timed auto-run as report *generators*. Add a **once-per-day guard** so a given day's report saves only once — whichever fires first (manual button, timed backup, or safety net) wins.
- **Non-negotiable — morning open must feel instant.** Render the page first; persist to Notion after. This is the entire point: Chris has ADD and a slow first load loses him. Local clear is instant (in-memory); only the Notion write is slow, and it happens behind the already-rendered screen.

### 2. Morning Routine → its own page — fixes "routine clutters dashboard all day"
- Move the routine **checklist items** off the dashboard onto a dedicated **Routine page**.
- **DO NOT touch the dashboard's existing morning-routine progress tracker.** Leave it exactly as-is. It keeps reading completion and stays visible all day as the "I hit my daily goal" confirmation. It is a passive readout (not clickable) and stays visible even at 100% as the win/trophy.

### 3. First-run routing — simple version
- The **first** app open of a new day (the reset run) lands on the **Routine page**.
- **Every other open** that day → dashboard. No "smart" routing — if Chris closes/reopens mid-morning before finishing, that's an "other open" and goes to the dashboard; he navigates to the routine himself. (Chris explicitly chose simple over smart for now.)

### 4. Brain Dump triage / Unsorted — fixes "posts go nowhere, no way to sort"
- Brain Dump stays the catch-all capture inbox (default home for new thoughts).
- Add **triage:** each item can be **promoted into one of the real task areas** OR **deleted**.
- **CRITICAL (see constraints):** any new entity created by promotion must be registered in the State-page id-arrays, or it vanishes on reload. This is the exact class of bug that bit v1.4.0 (`addRoutine`). Verify survival across a reload before calling it done.

### 5. Pomodoro Focus page — new, aligns the timer with its own purpose
Rationale: a timer on a busy dashboard contradicts Pomodoro's "one thing, nothing else in view." The dashboard timer becomes the doorway; the Focus page is the room.

- **Keep the dashboard timer where it is** — it becomes the entry point. Clicking it navigates to the Focus page.
- **Focus page is bare:** large timer, start/pause, a single "what am I focusing on" intention line, and a **"Park this thought"** button that dumps straight into Brain Dump (so a mid-session stray thought gets captured without breaking focus).
- **Timer runs off SHARED state** — the dashboard mini-timer and the Focus page must be the *same* running clock, not two copies. Start on either, glance at the other, same count.
- **Do NOT change** the focus-session tracking in the upper dashboard. It keeps logging sessions exactly as it does now.
