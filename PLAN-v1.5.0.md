# Command Center v1.5.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship five scoped UX/bug fixes — a reliable on-load daily reset, a dedicated Routine page, Brain Dump triage, and a bare Pomodoro Focus page — onto the live Netlify `dist` build.

**Architecture:** The app is built by `node cc-build.js`, which assembles `src/index.html` (markup template) + `src/styles.css` + `src/app.js` + `cc-data.js` (tested pure helpers) into `dist/index.html` via one atomic `writeFileSync`. Pure, testable logic lives in `cc-data.js` and is unit-tested with `node --test cc-data.test.js`; `src/app.js` is the browser IIFE that wires that logic to the DOM and is verified by building + reloading. This plan pushes every new piece of *logic* into `cc-data.js` (so it gets a failing-test-first treatment) and keeps `src/app.js` edits to thin DOM glue verified by reload.

**Tech Stack:** Vanilla ES5-style browser JS (IIFE, no framework), Notion MCP backend via `window.cowork.callMcpTool` (Cowork) or the Netlify `notion-proxy` function (browser), Node's built-in test runner for `cc-data.js`, Netlify GitHub-auto-deploy for production.

## ✅ Resolved decision — daily report persistence (Chris, this session)

START-HERE item 1 says the reset should "save the prior day's report to Notion in the background." **There is no Reports database in the Notion backend** (verified: State, Tasks, Wins, Routines, Capture, Focus Sessions — no Reports DB). The existing `openBrief()` is a *morning* brief shown in an `alert()`; nothing is persisted as an end-of-day report today.

**Decision: append the report to the Capture DB.** When a reset runs, the prior-day report is written to the Capture data source as one page — title `[Daily Report <YYYY-MM-DD>]`, the full report text in the `Notes` property, and `Processed:"__YES__"`. Because `liveLoad()` already skips captures with `Processed==="__YES__"` (app.js ~line 179), the report **never appears in Brain Dump / triage** (Phase 4). No schema change and no new id-list needed.

**Once-per-day guard stays instant:** a `localStorage` key `cc_report_<YYYY-MM-DD>` is still written and checked as the guard, so the on-load reset never blocks on a Notion read ("morning open must feel instant"). The only edge case: a *second device* opening for the first time after midnight has no local key, so it could write a duplicate `[Daily Report]` capture. That's a rare, harmless duplicate (both are `Processed`, hidden from triage); accepted for v1.5.0.

  function dailyReportText(dateStr, tasks, routines, wins){

```js
  // v1.5.0: fire on load when the stored reset date != today. Instant local clear,
  // background Notion writes. Returns true if a reset ran (used for first-run routing in Phase 3).
  function runDailyReset(){
    var today=todayHST();
    var last=app.state&&app.state.lastResetDate;
    if(!CCData.needsDailyReset(last,today)) return false;
    var prior=last||hstDate(new Date(Date.now()-864e5));
    // 1. safety-net report for the prior day (once per day): local guard + Capture-DB append
    if(!hasReport(prior)){
      var rpt=CCData.dailyReportText(prior, app.tasks, app.routines, app.wins);
      saveReport(prior, rpt);
      pushReportToNotion(prior, rpt);
    }
    // 2. capture which routines were checked, for the background Notion clear
    var toClear=app.routines.filter(function(r){return r.done;}).map(function(r){return r.id;});
    // 3. clear local state immediately (in-memory = instant)
    app.routines=CCData.clearedRoutines(app.routines);
    var o=lsGet(); o.pending=CCData.purgeRoutineOps(o.pending); lsSet(o);
    // 4. set the new reset date
    app.state.lastResetDate=today;
    // 5. background: persist new reset date + clear Done Today in Notion (do NOT await)
    if(app.mode==="live"){
      saveState({lastResetDate:today});
      toClear.forEach(function(id){
        call(T.update,{page_id:id,command:"update_properties",properties:{"Done Today":"__NO__"}}).catch(function(){});
      });
    }
    return true;
  }
```
